Case Study

Clicker Evolution

A custom moderation bot for a Discord server. Written from scratch in Node.js.

Project
Clicker Evolution
Role
Bot Developer
Stack
Node.js, discord.js
Commands
25+

The problem

The Clicker Evolution server needed moderation that could run without constant manual oversight. Existing bots handled the basics but not the specifics. Tiered punishment, burst-spam detection, and bans that persisted across restarts all required custom code.

The mod team was spending time on things automation should handle. The goal was to flip that ratio: let the bot cover the predictable patterns, and save human attention for the things bots can't do.

What was built

A moderation bot written from scratch in Node.js using discord.js. Dual slash and prefix command support across 25+ commands covering the full moderation workflow.

The punishment system escalates automatically based on accumulated warns. At 3 warns, a mute. At 7, a kick. At 10, a tempban. At 15, a permban. Each threshold is configurable. Tempbans write to disk so a bot restart doesn't free someone still serving time.

Automod watches for burst-spam patterns and acts before a human has to. Logging is split by purpose: mod actions go to one channel, automod events to another, so staff can monitor without noise.

Technical highlights

Tiered auto-punishment

Escalates from mute at 3 warns to permban at 15. Each threshold configurable per server.

Persistent tempbans

Ban records written to disk. Bot restarts don't reset active bans.

Burst-spam detection

Tracks message rate within a rolling window. Acts on threshold breach before humans notice.

Split logging

Mod actions and automod events go to separate channels. Clean signal, no noise.

Dual command support

Full slash command tree plus legacy prefix support. Works for both staff workflows.

25+ commands

Warn, mute, kick, ban, tempban, unban, purge, case lookup, and more.

The result

The mod team stopped manually handling most routine cases. The bot covers the predictable patterns. Staff attention goes to the things bots can't handle.

The server runs with less manual overhead than before. When something does need human review, it's in the right log channel, with context, not buried in a general feed.
